Version 0.6.1.0 Released: Quality of Life Improvements & Enhanced User Experience

September 12, 2025
Update

We're excited to announce Live Beats WebRadio Player version 0.6.1.0, a focused quality-of-life update that addresses user feedback and enhances the daily listening experience. This release fine-tunes key interface behaviors, optimizes system resource usage, and provides clearer user options for favorite station management.

Homepage Button Intelligence

The homepage button now properly clears when switching to stations without websites, eliminating the confusing behavior where previous station URLs would linger when the new station doesn't broadcast any website information. This ensures the homepage button always reflects the current station's actual web presence, providing accurate and relevant links every time.

Optimized Volume Settings

Volume settings are now saved only when the application exits, significantly reducing resource usage during playback sessions. Previously, volume changes were saved immediately, causing unnecessary disk operations during active listening. This optimization improves overall system performance while maintaining reliable volume persistence across sessions.

Enhanced Radio Browser Integration

Radio Browser stations now intelligently manage favorites selection - the favorites dropdown properly clears when playing non-favorite stations while correctly maintaining selection for stations that are actually in your favorites list. This prevents confusion about which playback mode is active and ensures accurate statistics tracking.

Improved User Interface Responsiveness

  • Non-Modal About Dialog: The About dialog is now non-modal, allowing you to continue using the main player interface while viewing version information and changelog details.
  • Enhanced Button Focus: Disabled buttons now bring focus to the interface instead of being completely unresponsive, providing better visual feedback and maintaining interface responsiveness even when certain functions are temporarily unavailable.

Clearer Favorite Station Playback

When playing Radio Browser stations that are already in your favorites, you now get clear, accurate options: play from favorites without reporting to the Radio Browser API, or play from favorites while also reporting the vote to the API. Both options ensure your listening statistics are properly tracked, with the choice simply determining whether to contribute data back to the Radio Browser community.

Version 0.6.1.0 builds on the solid foundation of 0.6.0.0 with these targeted improvements that make daily radio listening more intuitive and efficient. These refinements ensure every interaction feels natural and responsive while maintaining the robust stability and feature completeness that defines Live Beats WebRadio Player.

Rock-Solid Timer Stability

The Time Played timer has been completely overhauled and is now incredibly stable without any skipping or timing irregularities. This fundamental improvement ensures accurate tracking of your listening sessions and eliminates the frustrating timer jumps that could occur in previous versions.

We've resolved all scrolling issues affecting song titles, station names, and user messages. Text now flows smoothly across the interface without stuttering, jerking, or display artifacts, creating a much more polished and professional user experience.

A powerful new feature gives you complete control when a station exists in both your favorites and the radio browser. Critical Stream Handling Fixes

We've eliminated a potentially serious deadlock issue in stream reader disposal that could cause the entire application to freeze. The new implementation includes robust timeout protection (2 seconds) for all network stream cleanup operations and a completely improved disposal process that properly handles base stream closure, preventing hangs and ensuring smooth operation even with problematic network conditions.

Enhanced User Communication

User messages have been expanded throughout the application to provide better feedback for various operations. You'll now receive clear, informative messages that keep you informed about what the application is doing, from import operations to network status updates, making the entire experience more transparent and user-friendly.

Radio Browser Stability Improvements

  • Thread Safety: Completely resolved thread safety issues when populating radio browser filter dropdowns, eliminating crashes and unexpected behavior.
  • API Rate Limiting Fix: Solved duplicate radio browser form initialization that was causing API rate limiting issues, ensuring consistent access to station data.
  • Enhanced Loading Stability: Country, language, and tag loading in the radio browser is now significantly more stable and reliable.

Immediate Favorites Tracking

Fixed the frustrating issue where adding a station to favorites required restarting the station to begin proper tracking. Station tracking now immediately switches from non-favorite to favorite mode the moment you add a station to your favorites list. Additionally, imported favorite stations are now properly tracked without requiring an application restart, making the favorites system truly seamless.

Refined Import Experience

The settings panel will no longer unexpectedly open when performing drag-and-drop operations to import favorites files. The panel now only closes after import if it was already open before the operation, maintaining a cleaner and more predictable user interface behavior.

Polish & Professional Appearance

Removed the distracting focus rectangles that appeared on buttons when switching between applications, creating a cleaner, more professional appearance that maintains visual consistency regardless of application focus state.

Version 0.5.6.0 demonstrates our dedication to delivering enterprise-level stability and reliability in a consumer-friendly package. These improvements address the core issues that affect daily usage, ensuring Live Beats WebRadio Player provides the smooth, dependable experience you deserve.
