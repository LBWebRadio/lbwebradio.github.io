Version 0.8.0.0 Released: Major Database Overhaul & Enhanced Reliability

October 19, 2025
Major Update

Live Beats WebRadio Player version 0.8.0.0 represents a major leap forward in database reliability and data integrity. This release focuses on ensuring your listening history and favorites are always safely stored with extensive improvements to how the application handles database operations. We've also squashed several critical crashes and made meaningful refinements throughout the application.

Bulletproof Database Checkpoint System

The database checkpoint system has been completely rebuilt with automatic retry logic that ensures your data is always properly saved. When performing major database operations the system now attempts multiple retries if the initial save fails. This means your listening history and station data are protected even in edge cases where database writes might encounter temporary issues. The new system monitors the save process and won't proceed with critical operations until it confirms everything is safely written to disk.

Enhanced Database Lock Handling

We've added robust handling for database locks and busy states throughout the application. While these are edge cases that most users won't encounter the new protection ensures smooth operation even when multiple processes need database access simultaneously. The application now intelligently waits and retries rather than failing when the database is temporarily busy.

Safe Database Import Operations

A critical bug has been fixed where importing a database could cause corruption if WAL (Write-Ahead Logging) files were present in the database directory during the import. The import process now properly handles these temporary database files and ensures background processes complete their work before performing the import. This prevents any potential data corruption and makes database restoration completely reliable.

Complete Database Export

Database exports now force a complete checkpoint before creating the backup file. This ensures that all recent changes are included in the export even if they haven't been written to the main database file yet. When you export your database you can be confident that absolutely everything up to that exact moment is captured in the backup. No more wondering if your most recent listening session made it into the export.

Fail-Safe Shutdown Process

The application shutdown sequence has been significantly enhanced with retry logic for database saves. When you close Live Beats WRP the system now makes multiple attempts to ensure all your data is properly saved before the application exits. While there were no known issues with data loss before this additional layer of protection provides peace of mind that your listening history and settings are always preserved. The shutdown process won't complete until it confirms everything is safely written to disk.

Reduced Risk of Data Loss

All these database improvements work together to dramatically reduce any risk of data loss during normal operation. Whether you're importing a backup or shutting down the application or experiencing unexpected system conditions your data remains protected. The combination of retry logic and careful checkpoint management means your listening history and favorites are safer than ever.

Clearer Database Operation Messages

We've improved the information messages displayed during database import and export operations. The messages now clearly explain what's happening at each step and provide better context about the process. This helps you understand what the application is doing with your data and removes any uncertainty about database operations.

Preserved Database Creation Date

The database creation date is now properly preserved when clearing error logs from the database information viewer. Previously this date would be reset when clearing the log data. The creation date is now treated as permanent metadata that survives maintenance operations.

Critical Crash Fixes

This release fixes a crash that could occur when closing the Advanced Radio Browser if the main application was already in the shutdown process. The browser window now detects when shutdown is in progress and closes gracefully without attempting operations on resources that are being cleaned up. We've also eliminated a rare null reference crash in a background task that could occur under specific timing conditions.

Auto-Stop Protection Fix

An issue has been resolved where unmuting the player wouldn't properly reset the auto-stop protection timer. The 30-minute muted playback protection now correctly resets when you unmute ensuring the feature works as intended. You won't experience unexpected stops after unmuting if you had previously been playing muted.

Enhanced Station Information Display

The database information and error log viewer now displays the full stream URL alongside each monitored station name. Previously only station names were shown which made troubleshooting difficult if an incorrect URL format made its way into the favorites system. The complete URL display helps identify any problematic links and provides better insight into which exact streams are being tracked in your database.

Improved Station Website Button

The station website button now includes intelligent filtering that removes invalid and placeholder URLs before attempting to open them. We've enhanced the URL validation with better protection against malformed links that could cause errors. The button only activates for legitimate website addresses ensuring you never encounter broken links or error dialogs.

Smarter Random Station Selection

The play random station button in the favorites search window now avoids selecting the same station consecutively. If you're already listening to a station and press the random button it will pick something different from your filtered results. This makes the random feature more useful for discovering other stations in your collection.

Performance Improvements

Data usage tracking has been optimized for better stability and performance. We've also cached the regex pattern used for bitrate parsing which eliminates wasteful object recreation on every parse operation. These behind-the-scenes optimizations contribute to smoother overall performance.

Intuitive Label Change

The "Time Played" label shown when connected to a station has been renamed to "Streaming". This more accurately describes what's happening and feels more intuitive. You're streaming the station rather than playing back a fixed duration of content.

Manual Temporarily Unavailable

Please note that the Live Beats manual is currently disabled while we work on a completely new version. The updated manual is under construction and will be available soon with comprehensive coverage of all the latest features and improvements.

Version 0.7.6.0 Released: Performance Optimization & Smart Database Maintenance

Automatic Database Optimization

Perhaps the most significant addition in this version is the self-maintaining database system. Every 7 days, Live Beats automatically optimizes its database to maintain peak performance and responsiveness throughout extended usage.

Behind the Scenes: How Your App Stays Fast and Healthy

Ever notice how some apps start fast when you first install them, but over time they get slower, heavier, or even a little buggy? That usually happens because the app's database — the "brain" where all your information is stored — gets cluttered and needs cleaning.

Live Beats WRP has a built-in self-maintenance system that takes care of this automatically, without you ever needing to push a button. Here's how it works, behind the scenes:

1. A Quick Check When You Start Listening

When you play a radio station for the first time in a session, Live Beats runs a quick internal check. This is like a warm-up routine — making sure everything is working correctly before you keep listening.

2. Weekly Database Maintenance

Once that first check is done, the app also looks at when it last performed database maintenance. If it's been more than seven days, it immediately starts a background cleanup.

This includes:

  • Tidying up unused space so the database doesn't bloat unnecessarily
  • Reorganizing data so it can be read more quickly
  • Refreshing performance statistics so that Live Beats knows the best way to retrieve your information

Think of it like reorganizing a messy filing cabinet — your documents are still there, but now they're easier and faster to find. And don't worry: all your favorites, themes, and settings remain safely saved during this process.

3. No Interruptions, No Popups

All of this happens silently. The app won't interrupt your music with popups or technical warnings. If maintenance is needed, it runs quietly in the background while you enjoy your radio station. Once it's complete, you'll see a small success message (about once every 7 days) letting you know the database has been optimized. This process was designed with you in mind: minimal impact and maximum benefit.

4. Smarter Scheduling

The app keeps track of the last time maintenance was done. If it's too soon to run again, it waits — ensuring the process only happens when it's actually needed.

5. Manual Maintenance (Optional)

If you ever want to run database maintenance yourself, you can do so from the Database window in the app. Most users will never need this since the process happens automatically in the background — but if you want to, running it manually won't hurt anything. It's just an extra tune-up on demand.

Frequently Asked Questions

Do I need to run this myself?

Nope! Maintenance happens automatically every 7 days. The manual option is there if you want to use it, but it's not required.

Will this slow down my computer while I'm listening?

Not noticeably. The process is designed to be quick and silent. In most cases, you won't even know it's happening.

Why does it trigger when I play a radio station?

Because that's the moment the app knows you're actively using it. It's the perfect time to check if maintenance is due without slowing things down during loading.

What's the benefit for me as a listener?

Faster load times, smoother playback, and a database that doesn't grow unnecessarily large. Basically, your app stays just as responsive as the day you first installed it.

What if something goes wrong during maintenance?

The process is designed to fail silently — meaning if there's ever a rare issue, the app won't interrupt your listening experience.

Do I need to wait until maintenance is done before I exit Live Beats?

No. If you close the app during maintenance, it will simply stop and pick up again later when needed. You don't have to keep the app open for it to complete.

In short: the next time you hit "play" on your favorite radio station, know that the app is quietly working behind the scenes to keep itself fast, clean, and optimized — and if you ever feel like giving it a little extra boost, you can run maintenance manually anytime from the Database window.
